# Vertex AI Search との比較

## Vertex AI Search とは

Google Cloud のマネージドな検索サービス。文書をアップロードするだけで、チャンク分割・Embedding・インデックス構築・検索が自動で行われる。独自パイプラインで手作業していた工程の大部分が不要になる。

## セットアップ

| 工程 | 独自RAG | Vertex AI Search |
|------|---------|-----------------|
| チャンク分割 | 自前実装 | **自動** |
| Embedding生成 | Vertex AI API呼び出し | **自動** |
| インデックス構築 | Firestoreベクトルインデックス手動作成 | **自動** |
| 検索ロジック | ハイブリッド検索+RRF統合を自前実装 | **自動（組み込み済み）** |
| **所要時間** | **約1.5日**（段階的改善含む） | **1時間未満** |

## 実測比較（同一評価データセット）

同じ74件のテスト質問、同じLLM-as-Judge判定で直接比較した。

## 考察

### 検索基盤としてVertexは十分

チューニングなし・チャンク最適化なしで84.4%。独自RAGが28.4%から約1.5日かけて到達した水準に、1時間未満で到達している。

### 得意・不得意が異なる

- **Vertexが強い**: 完全一致検索（100%）、意味検索（66.7%）、新旧文書の区別（100%）
- **独自RAGが強い**: 似た番号の区別（100%）、複数文書の統合（100%）、表の読み取り（100%）

これはチャンク分割の方式の違いに起因する。独自RAGはContextual Retrieval（LLMによる文脈付与）を実施しており、ドメイン固有の情報を保持しやすい。

### 独自RAGの運用コスト

独自RAGは精度を出せる一方、構築・運用に大きな手間がかかる:

| 工程 | 具体的な手間 |
|------|------------|
| チャンク分割 | 文書の特性に合わせたサイズ・重複幅の調整。最適値を見つけるために複数パターンで実験が必要だった |
| Contextual Retrieval | チャンクごとにLLMを呼んで文脈説明を付与。61ファイルで約25分、文書が増えれば比例して増加 |
| ハイブリッド検索 | ベクトル検索+キーワード検索+RRF統合を自前実装。型番抽出ロジックや日本語トークナイズの調整が必要 |
| バグの発見・修正 | 評価パイプライン自体のバグ（権限フィルタ無効等）を4段階で発掘。マネージドサービスでは発生しない類のバグ |

Vertex AI Search はこれらが全て不要であり、**精度が同等であれば本番運用ではVertexを選択するのが合理的**。

### 本番での推奨

本番運用では検索基盤はどちらかを選択することになる。PoCの結果を踏まえた推奨:

| 条件 | 推奨 |
|------|------|
| 運用保守の負荷を下げたい | **Vertex AI Search** — 検索基盤の運用をGoogleに委任 |
| ドメイン固有のチューニングが必要 | **独自RAG** — チャンク分割やスコアリングを細かく制御可能 |
| まだ判断できない | PoCでは両方を同じ評価基盤で比較できる実装が完了しており、本番データで再比較して決定可能 |
