# 申し送り — 本番データで取り組む際の注意点

## 評価の進め方

### 簡易評価と本格評価の2層化を推奨

PoCでは74件のテスト質問を毎回フル実行していたが、1回あたりのAPI費用が蓄積し、計測コストが問題になった。

| 評価レベル | 件数 | 用途 | 所要時間 |
|:---:|:---:|------|:---:|
| **簡易評価** | 10〜15件 | パラメータ変更の効果確認、方向性の判断 | 5分程度 |
| **本格評価** | 全件 | マイルストーンでの正式スコア計測 | 30分程度 |

本番では、まず簡易評価セット（各カテゴリから代表1件ずつ）を定義し、日常的な調整には簡易版を使う。本格評価はマイルストーン時のみ実行する運用が望ましい。

---

## PoCで繰り返し発生した問題

同じ問題が形を変えて何度も発生した。本番では事前に対策しておくべき:

### 1. 評価の仕組み自体のバグ

**何が起きたか**: 評価プログラムが権限フィルタを無効にした状態で動いていた。最上流（評価スクリプトがRAGに権限情報を渡していない）のバグが、4段階の修正を経てようやく発見された。

**本番での対策**: 評価を始める前に「評価の仕組み自体が正しく動いているか」を確認するチェック項目を用意する。特に権限制御のように「何も起きないことが正しい」機能は、テストが正しく動いていないと見逃しやすい。

### 2. LLMの空レスポンス

**何が起きたか**: Geminiの出力トークン上限が小さいと、AIの内部思考（thinking tokens）で枠を使い切り、回答が空になる問題が4回再発した。

**本番での対策**: 出力トークン上限は十分大きく設定する（8192以上）。空レスポンスが発生したら、まずトークン上限を疑う。

### 3. メタデータの不備

**何が起きたか**: 文書のメタデータ（権限レベル等）がYAML形式のパースミスで壊れており、権限フィルタが効かない文書があった。

**本番での対策**: 文書をアップロードする前に、メタデータの形式・値が正しいかを自動チェックする仕組みを入れる。特に権限情報は「壊れていても動いてしまう」ため、目視では発見しにくい。

## 「最初からこうすればよかった」

| やったこと | 振り返り |
|-----------|---------|
| 74件のテストを毎回フル実行 | 簡易評価（10件）で傾向を掴んでから本格評価すべきだった |
| チャンクサイズを根本原因と予測して実験 | 先にEmbeddingの設定（task_type）やノイズ比率を分析すべきだった |
| 複数の改善を同時に適用して計測 | 変数は1つずつ変えないと、何が効いたか分からない |
| バグ修正と精度計測を並行で進めた | 評価基盤が正しく動くことを先に保証すべきだった |
| 新しいLLMモデルに期待して切り替え | モデル性能はボトルネックではなかった。切り替え前に仮説を検証すべきだった |

## 本番データ固有の注意点

PoCは模擬データだったため発生しなかったが、本番では注意が必要なこと:

### 権限制御の設計

PoCでは2段階（公開 / 管理職のみ）の模擬権限で検証した。本番では:
- 部署・役職の組み合わせが複雑になる
- 「公開文書」の定義が曖昧だと、全ユーザーに見えるべき文書が見えなくなる
- 権限チェックは検索パイプラインの**全段階**（ベクトル検索・キーワード検索・ハイブリッド統合）で漏れなく実装する必要がある。PoCでは1箇所の漏れで機密文書が混入するバグが発生した

### 文書の多様性

PoCの文書は19件のMarkdownで、構造が均質だった。本番では:
- PDF、Word、Excel など形式が混在する可能性
- 図表や画像を含む文書への対応
- 文書のバージョン管理（古い版と最新版の区別）

### ノイズの特性

PoCではWikipedia記事を均質なノイズとして投入した。本番では:
- 関連性が高いが間違った文書（古い版、別部署の類似規程）がノイズになる
- PoCとは異なるノイズ対策が必要になる可能性がある

## 本番で最初にやるべきチェックリスト

本番データを受領したら、精度改善に着手する前に以下を確認:

- [ ] 文書のメタデータ（権限レベル・カテゴリ・更新日）が正しく設定されているか
- [ ] 評価スクリプトが権限情報を正しくRAGに渡しているか
- [ ] 簡易評価セット（10〜15件）を定義したか
- [ ] 簡易評価で「そもそもRAGが動いているか」を確認したか
- [ ] LLMの出力トークン上限が十分か（8192以上）
- [ ] Firestoreを使う場合、ベクトルインデックスが作成されているか
