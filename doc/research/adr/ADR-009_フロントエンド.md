# ADR-009: フロントエンド

- **ステータス**: 採用
- **日付**: 2026-03-20

## 選択肢

| # | 選択肢 | 気軽さ | コスト | 数値評価 |
|---|--------|--------|--------|---------|
| 1 | **Firebase Hosting** | **◎ `firebase deploy` で即公開** | **◎ 無料枠 10GB/月** | — |
| 2 | Cloud Run | ○ Dockerfile必要 | 従量課金 | — |
| 3 | 既存社内ポータル組込 | △ 既存システムの改修が必要 | 既存費用内 | — |

## 決定

**Firebase Hosting** を採用。

## 理由

- `firebase deploy` 一発でデプロイでき、最も気軽
- 無料枠（ストレージ10GB、転送量360MB/日）でPoCには十分
- Cloud Run にAPIをホストし、Firebase Hosting はフロントの静的配信に特化する構成で、責務が明確

## 影響

- SSR（サーバーサイドレンダリング）が必要になった場合は Cloud Run との併用を検討
