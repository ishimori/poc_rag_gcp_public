# ADR-008: フレームワーク

- **ステータス**: 採用
- **日付**: 2026-03-20

## 選択肢

| # | 選択肢 | 気軽さ | コスト | 数値評価 |
|---|--------|--------|--------|---------|
| 1 | **Genkit** | **◎ npm install で即開始** | **◎ OSS無料** | **◎ Evaluator + Trace が組込み** |
| 2 | LangChain | ◎ | ◎ OSS無料 | △ 評価は別途 Ragas 等が必要 |
| 3 | LlamaIndex | ○ | ◎ OSS無料 | △ 同上 |

## 決定

**Genkit** を採用。

## 理由

- **数値評価の決め手**: Evaluator（自動採点）と Trace（推論過程の可視化）が標準搭載されており、「チューニング効果がブラックボックス化しない」という要件に最も直接的に応える
- Firebase（Auth / Firestore / Hosting）とネイティブ統合されており、追加の接着コードが最小限
- `npx genkit start` で開発UIが即起動し、Flow の実行・トレース確認がブラウザでできる

## 影響

- チャンキング処理（Python系ライブラリが充実）はGenkit外で実装し、結果をFirestoreに格納する構成になる
