# ADR-002: チャンキング方式

- **ステータス**: 採用
- **日付**: 2026-03-20

## 選択肢

| # | 選択肢 | 気軽さ | コスト | 数値評価 |
|---|--------|--------|--------|---------|
| 1 | 固定長分割 | ◎ 最もシンプル | 無料 | ○ |
| 2 | **Recursive Character Text Splitter** | **◎ LangChain で1行** | **無料（ローカル処理）** | **◎ パラメータ変更 → Evaluator で比較** |
| 3 | セマンティックチャンキング | △ Embedding APIコール必要 | APIコスト発生 | ◎ |

## 決定

**Recursive Character Text Splitter**（LangChain）を採用。

## 理由

- `pip install langchain-text-splitters` だけで即使える
- チャンクサイズ・オーバーラップをパラメータで変えるだけでA/Bテストができ、Evaluator で Faithfulness / Context Precision を数値比較可能
- APIコールが不要でローカル処理のみ（コストゼロ）
- セマンティックチャンキングは精度が上がらなかった場合の次の手段として温存

## 影響

- 日本語向けセパレータ（`。`、`、`）の調整が必要
- ヘッダー・インジェクションは別途実装する
