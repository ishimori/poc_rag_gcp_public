# 補足: チャンク調整の検証手法

> 「なんとなく良くなった気がする」という感覚頼みの調整を、科学的な「実験と検証」に変えるためのツールと手法。

---

## 1. Firebase Genkit の「Evaluators」による自動評価

Firebase Genkit には、RAGの精度を定量化するための **Evaluators（エバリュエーター）** 機能が組み込まれている。

### 評価指標

AI（Gemini）が「評価者（Judge）」となり、RAGの回答を以下の指標で採点する。

* **Faithfulness（忠実性）**: 回答の根拠が、検索されたチャンクの中に本当にあるか？（ハルシネーションの検知）
* **Answer Relevancy（回答の関連性）**: ユーザーの質問に対して、的確に答えているか？
* **Context Precision（コンテキスト適合率）**: 検索された上位チャンクの中に、本当に正解に繋がる情報が含まれていたか？

### ワークフロー

1.  **評価用データセットの作成**: 「質問」と「期待される正解（Ground Truth）」のペアを30〜50個ほど用意する。
2.  **複数のチャンク設定でFlowを実行**: 「チャンク500文字・重複100」と「チャンク1000文字・重複200」など、設定を変えた2つのRAG Flowを作る。
3.  **Genkit UI で比較**: GenkitのDeveloper UI上でこれらを実行すると、各設定の「平均スコア」がグラフ化される。

## 2. Vertex AI「Rapid Evaluation SDK」での大規模検証

Genkitが開発中の試行錯誤に向いているのに対し、本番環境に近い大規模なデータで「どのチャンク戦略が最強か」を決めるのが **Vertex AI の評価サービス**。

### 実装のポイント

* **プログラマブルな評価**: Python SDK を使い、数百件のテストケースを一括で走らせる。
* **カスタム・ルーブリック（評価基準）**: 「ネジ番号の回答は、形式が `[XXXXXX]` でなければならない」といった独自の採点基準をAIに渡し、厳密に評価させることができる。

```python
# Vertex AI SDKでの評価実行イメージ
from google.cloud import aiplatform

# 複数のチャンク設定をバッチで評価
eval_task = aiplatform.RapidEvaluation(
    dataset="gs://my-eval-data/questions.jsonl",
    metrics=["groundedness", "fulfillment", "summarization_quality"]
)
results = eval_task.evaluate()
```

## 3. 「Ragas」との連携

RAG評価のオープンソース・フレームワークである **Ragas** は、Google Cloud（Vertex AI / Gemini）とも相性が良い。

* **RAG Triad（RAGの三要素）**: 「質問・チャンク・回答」の三者の関係性を数値化する。
* **自動問題作成**: 手元にある大量のPDF（チャンク）から、**AIが自動で「試験問題」を作る機能**がある。これにより、人間がテストケースを作る手間を省き、チャンク設定を変えるたびに即座に「合格率」を出せる。

---

## 4. 調整作業の「システム化」手順

1.  **ベースラインを決める**: 今の「精度が出ない」状態をまず数値化する。
2.  **A/Bテスト**: チャンクサイズや分割アルゴリズムを変えた「候補」を複数作る。
3.  **自動評価**: SDKを実行し、スコア（Faithfulness等）が上がったものだけを採用する。
4.  **トレースの確認**: スコアが低いケースだけを Genkit の Trace 機能で開き、「なぜこのチャンクではダメだったのか」を深掘りする。

### 要点

チャンク調整は「パラメータいじり」ではなく、**「データに対するテストコードを書く」**という感覚に近い。Googleのツール群は、その「テスト」をAIに代行させるための枠組みを提供している。
