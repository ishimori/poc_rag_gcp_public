# 補足: コスト試算

> 3,000人規模の企業でGoogle Cloud（Firebase）ベースのRAGを運用する場合のコスト算出ロジック。

---

## 1. 初期費用（Initial Cost）

データ投入（インジェスト）時にかかるコスト。既存のドキュメント量に比例する。

| 項目 | サービス | 算出目安 | 備考 |
| :--- | :--- | :--- | :--- |
| **ドキュメント解析** | Document AI (Layout Parser) | 約 **$30〜$50** / 1,000ページ | 表構造やレイアウトを維持して抽出する際のメインコスト |
| **ベクトル化** | Vertex AI Embedding API | 約 **$0.0001** / 1,000文字 | 非常に安価。5,000万文字でも$5程度 |
| **データ蓄積** | Cloud Storage (GCS) | 約 **$0.02** / GB | 生データと中間データの保存用 |

**試算例** — 社内に10,000ページのPDF/Excel資料がある場合:

* Document AI: 約 $500
* Embedding: ほぼ無視できるレベル
* **合計: 約7〜8万円程度**（初回のみ）

---

## 2. 月次運用費用（Monthly Running Cost）

3,000人の社員のうち、毎日10%（300人）が3回質問すると仮定（月間約27,000リクエスト）。

| 項目 | サービス | 算出目安 | 月間コスト(目安) |
| :--- | :--- | :--- | :--- |
| **LLM推論 (Input/Output)** | Gemini 1.5 Flash | $0.075 / 1M tokens | **約 $5〜$10** |
| **LLM推論 (Complex)** | Gemini 1.5 Pro | $1.25 / 1M tokens | **約 $50〜$100** |
| **再順位付け (Reranking)** | Vertex AI Ranking API | **$1.00** / 1,000 requests | **約 $27** |
| **ベクトル検索** | Firestore Vector Search | 読み書き $0.06 / 10万回 | **約 $5** |
| **ロジック実行** | Cloud Functions / Run | 実行時間ベース | **約 $10〜$20** |

**合計（月間クラウド利用料）**:

* Gemini 1.5 Flashメイン: **約1〜2万円 / 月**
* Gemini 1.5 Proメイン: **約2〜3万円 / 月**

---

## 3. コスト最適化のポイント

### モデル・ルーティング

質問の難易度を自動判定し、簡単な質問（ネジ番号の照会など）は **Flash**、複雑なトラブル相談は **Pro** に振り分けることで、コストを30〜50%削減できる。

### Document AIのキャッシュ戦略

一度解析したPDFの結果は必ずGCSにJSON形式で保存（キャッシュ）する。「プロンプトを少し変えて試したい」という時にDocument AIを再度叩くと数万円単位でコストが跳ね上がるため、**「解析済みデータ」を再利用するパイプライン設計**が保守コストを抑える鍵。

### 開発人件費

クラウド利用料よりも、開発エンジニアの工数が最大のコスト。**Genkit** を使い、AIエディタで定型コードを自動生成させることで、開発期間を短縮しトータルコストを抑える。

---

## まとめ: 3,000人規模での予算感

* **初期構築（クラウド実費）**: 10〜20万円（データ量による）
* **月次運用（クラウド実費）**: 2〜5万円
* **開発・保守（エンジニア工数）**: 数人月〜（ここが最大）

3,000人の会社で月数万円のシステム維持費は、人件費削減（問い合わせ対応時間の短縮）の効果を考えれば、極めてROIが高い。
