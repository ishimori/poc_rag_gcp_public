# 補足: コスト試算

> 3,000人規模の企業でGoogle Cloud（Firebase）ベースのRAGを運用する場合のコスト算出ロジック。

!!! warning "料金は変動します"
    以下の料金は2026年3月時点の参考値です。最新の料金は各公式ページを参照してください。

---

## 1. 初期費用（Initial Cost）

データ投入（インジェスト）時にかかるコスト。既存のドキュメント量に比例する。

| 項目 | サービス | 算出目安 | 備考 |
| :--- | :--- | :--- | :--- |
| **ドキュメント解析** | [Document AI (Layout Parser)](https://cloud.google.com/document-ai/pricing) | 約 **$10** / 1,000ページ | 表構造やレイアウトを維持して抽出する際のメインコスト |
| **ベクトル化** | [Vertex AI Embedding API](https://cloud.google.com/vertex-ai/generative-ai/pricing) | 約 **$0.000025** / 1,000文字 | 非常に安価。5,000万文字でも約$1.25 |
| **データ蓄積** | [Cloud Storage (GCS)](https://cloud.google.com/storage/pricing) | 約 **$0.02** / GB | 生データと中間データの保存用 |

**試算例** — 社内に10,000ページのPDF/Excel資料がある場合:

* Document AI: 約 $100
* Embedding: ほぼ無視できるレベル
* **合計: 約1.5〜2万円程度**（初回のみ）

## 2. 月次運用費用（Monthly Running Cost）

3,000人の社員のうち、毎日10%（300人）が3回質問すると仮定（月間約27,000リクエスト）。

| 項目 | サービス | 算出目安 | 月間コスト(目安) |
| :--- | :--- | :--- | :--- |
| **LLM推論 (高速)** | [Gemini 2.5 Flash](https://cloud.google.com/vertex-ai/generative-ai/pricing) | 入力 $0.15 / 出力 $0.60 / 1M tokens | **約 $10〜$20** |
| **LLM推論 (高精度)** | [Gemini 2.5 Pro](https://cloud.google.com/vertex-ai/generative-ai/pricing) | 入力 $1.25 / 出力 $10.00 / 1M tokens | **約 $80〜$150** |
| **再順位付け (Reranking)** | [Vertex AI Ranking API](https://cloud.google.com/generative-ai-app-builder/pricing) | **$1.00** / 1,000 requests | **約 $27** |
| **ベクトル検索** | [Firestore](https://cloud.google.com/firestore/pricing) | 読み $0.06 / 書き $0.18 / 10万回 | **約 $5〜$10** |
| **ロジック実行** | Cloud Functions / Run | 実行時間ベース | **約 $10〜$20** |

**合計（月間クラウド利用料）**:

* Gemini 2.5 Flashメイン: **約1〜2万円 / 月**
* Gemini 2.5 Proメイン: **約3〜5万円 / 月**

!!! note "Gemini 3.1 Pro について"
    2026年3月時点で Gemini 3.1 Pro がリリースされている。エージェント型ワークフローやコーディングに最適化されており、用途によってはこちらも選択肢になる。料金は公式ページで確認のこと。

## 3. コスト最適化のポイント

### モデル・ルーティング

質問の難易度を自動判定し、簡単な質問（ネジ番号の照会など）は **Flash**、複雑なトラブル相談は **Pro** に振り分けることで、コストを30〜50%削減できる。

### Document AIのキャッシュ戦略

一度解析したPDFの結果は必ずGCSにJSON形式で保存（キャッシュ）する。「プロンプトを少し変えて試したい」という時にDocument AIを再度叩くとコストが積み上がるため、**「解析済みデータ」を再利用するパイプライン設計**が保守コストを抑える鍵。

### 開発人件費

クラウド利用料よりも、開発エンジニアの工数が最大のコスト。**Genkit** を使い、AIエディタで定型コードを自動生成させることで、開発期間を短縮しトータルコストを抑える。

---

## まとめ: 3,000人規模での予算感

* **初期構築（クラウド実費）**: 2〜5万円（データ量による）
* **月次運用（クラウド実費）**: 1〜5万円（モデル選択による）
* **開発・保守（エンジニア工数）**: 数人月〜（ここが最大）

3,000人の会社で月数万円のシステム維持費は、人件費削減（問い合わせ対応時間の短縮）の効果を考えれば、極めてROIが高い。
