# DD-021-2: クエリログへの技術状態記録

> **経緯**: DD-021（検索プリセット）からログ記録部分のみ縮小・独立。プリセットUI部分は破棄。

## 目的

クエリログに「どの検索技術がONだったか」を記録し、回答の再現性・分析可能性を確保する。

## 背景・課題

### 現状

`main.py` `_save_query_log()` が保存するフィールド:
- `query`, `answer`, `model`, `elapsed_ms`, `sources`, `source_count`, `no_answer`, `timestamp`

### 問題点

- どの技術がONだったか不明 — 同じクエリでも設定が違えば結果が変わるが、ログからは区別できない
- 評価結果との対応付けが困難

## 検討内容

### 記録するフィールド

`_save_query_log()` に `techniques` フィールドを追加:

```python
"techniques": {
    "hybrid_search": True,
    "metadata_scoring": True,
    "clarification": True,
    "permission_filter": True,
    "shadow_retrieval": True,
    "multi_query": False,
    "contextual_retrieval": True,
}
```

### 値の取得元

UIは技術トグル変更時に `PUT /api/admin/config` でサーバー側の `config` シングルトンを更新する方式。
chat APIのリクエストボディには技術フラグを**送っていない**。
→ `_save_query_log()` 時点の `config` オブジェクトから直接取得すればよい。引数追加は不要。

## 決定事項

- `config` シングルトンから技術状態を取得（引数追加不要）
- ログ肥大化: 問題なし。7フィールドの bool 値 = 数十バイト。現状のログ1件あたり数KBに対して無視できる
- 後方互換: Firestore はスキーマレスなので、フィールド追加で既存ログは壊れない。`Logs.tsx` 側で `techniques` が undefined の場合は非表示にする
