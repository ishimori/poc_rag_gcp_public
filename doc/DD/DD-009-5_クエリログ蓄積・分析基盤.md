## 目的

RAGチャットの実行ログ（質問・回答・応答時間・使用モデル等）をFirestoreに蓄積し、ナレッジギャップの発見・評価データセット生成・パフォーマンス分析を可能にする。

## 背景・課題

- DD-009-3でTypeScript UIに移行し、チャット機能が動作している
- しかし「どんな質問がされたか」「どの質問に答えられなかったか」の記録がない
- PoCの価値を示すには「使われた結果」のデータが不可欠
- 蓄積されたクエリは評価データセット（DD-007）の拡充にも活用できる

## 検討内容

### 蓄積するデータ

| フィールド | 型 | 説明 |
|-----------|-----|------|
| query | string | ユーザーの質問文 |
| answer | string | LLMの回答文 |
| model | string | 使用したモデル名 |
| elapsed_ms | number | 応答時間（ミリ秒） |
| sources | array | リランク後にヒットしたソース一覧 |
| source_count | number | ヒットしたソース件数 |
| no_answer | boolean | 「記載がありません」回答だったか |
| timestamp | timestamp | 記録日時 |

### 保存先

- Firestore `query_logs` コレクション（既存のFirestoreをそのまま活用）
- Cloud Function内で回答生成後に非同期で書き込み（レスポンス遅延を防ぐ）

### 活用方法

1. **ナレッジギャップ分析** — `no_answer=true` のクエリを集めて、不足ドキュメントを特定
2. **評価データセット自動生成** — 実際の質問を `test-data/` に取り込み
3. **モデル比較** — 同じ質問に対するモデル別の回答品質・速度を比較
4. **利用傾向の可視化** — DD-009-4（管理者UI）で表示
