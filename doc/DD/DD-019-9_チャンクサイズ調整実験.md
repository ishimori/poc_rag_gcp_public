# DD-019-9: チャンクサイズ調整実験

## 目的

chunk_size / chunk_overlap のパラメータを変更してフル評価を実行し、最適なパラメータを特定する。DD-019-8 の分析結果を踏まえ、チャンキングが原因の失敗ケースに対して改善を図る。

## 背景・課題

現在のパラメータ:
- `chunk_size: 800`（文字）
- `chunk_overlap: 150`（文字）
- `contextual_retrieval: True`

グループBフル評価で Overall 50.0%。DD-019-8 で semantic 失敗の原因を分析した結果を踏まえて、チャンクサイズの調整が有効かどうかを判断する。

### チャンクサイズの影響

| パラメータ | 小さく（例: 400） | 大きく（例: 1200） |
|---|---|---|
| 検索精度 | 上がる（ノイズが減る） | 下がる（無関係な情報が混入） |
| 文脈の豊富さ | 下がる（情報が分断される） | 上がる（1チャンクで完結しやすい） |
| チャンク数 | 増える（コスト増） | 減る（コスト減） |
| step_sequence | 下がる可能性（手順が分断） | 上がる可能性（手順が1チャンクに） |
| table_extract | 下がる可能性（表が分断） | 上がる可能性（表が1チャンクに） |

### DD-019-8 との関係

DD-019-8 で「ベクトル検索段階で正しい文書が引けない」と判明した場合、チャンクの内容（文脈の付き方）やサイズが原因の可能性がある。DD-019-8 の結果次第で本DDの計画を調整する。

## 検討内容

### 実験計画

Tuning画面の「一括実行」機能（Save → Ingest → Evaluate）を活用し、複数パラメータでフル評価を実行する。

| 実験 | chunk_size | chunk_overlap | 仮説 |
|------|-----------|--------------|------|
| baseline | 800 | 150 | 現状（50.0%） |
| A | 600 | 100 | 小さいチャンクで検索精度向上 |
| B | 1200 | 200 | 大きいチャンクで文脈豊富に |
| C | DD-019-8の結果次第 | — | 原因に応じたパラメータ |

### 評価方法

- 各実験でフル評価（74件）を実行
- Tuning画面の History で結果を比較
- テストタイプ別の変動を確認（特に semantic, step_sequence, table_extract）
