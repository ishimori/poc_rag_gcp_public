# DD-024-2: クエリログの「動作結果」記録とUI改善

| 作成日 | 更新日 | ステータス |
|--------|--------|------------|
| 2026-03-22 | 2026-03-22 | 進行中 |

## 目的

クエリログのTechniques表示を「設定値の羅列」から「何が起きたか（動作結果）」に変え、ログを見る人にとって意味のある情報にする。

## 背景・課題

### 現状の問題

Admin画面のLogs → Detail パネルに表示される Techniques:

```
✓ shadow_retrieval  ✓ clarification  ✗ multi_query  ✓ contextual_retrieval
✓ metadata_scoring  ✓ hybrid_search  ✓ use_vertex_ai_search  ✓ permission_filter
```

これは**設定値（何がONだったか）**であり、**何が起きたか**ではない。

- Vertex使用時に `hybrid_search: true` が表示されても無意味（Vertex経由では使われない）
- `clarification: true` だけでは「聞き返しが発動したのか、しなかったのか」がわからない
- `shadow_retrieval: true` だけでは「権限拒否が起きたのか」がわからない

### ログを見る人が知りたいこと

| 関心 | 現状 | あるべき姿 |
|------|------|-----------|
| 何で検索した？ | Engine列 ✅ | 解決済み |
| 回答できた？ | No Answer列 ✅ | 解決済み |
| 遅くない？ | Time列 ✅ | 解決済み |
| 何を参照した？ | Sources ✅ | 解決済み |
| **なぜこの結果になった？** | Techniques（設定値） ❌ | **動作結果を記録** |

## 検討内容

### バックエンド: `_save_query_log()` に動作結果を追加

`rag_flow()` の `RAGResponse` には既に `is_clarification` がある。これに加え、以下の動作結果を記録する:

| フィールド | 型 | 意味 | 取得元 |
|-----------|-----|------|--------|
| `search_engine` | string | `"vertex"` or `"firestore"` | `config.use_vertex_ai_search` |
| `search_hit_count` | int | 検索ヒット件数 | `len(rag_result.sources)` |
| `rerank_count` | int | リランク後の件数 | `len(rag_result.reranked_sources)` |
| `is_clarification` | bool | 聞き返しが発動したか | `rag_result.is_clarification` |
| `is_permission_denied` | bool | 権限拒否が発動したか | 回答文に権限拒否マーカーが含まれるか |

### フロントエンド: Detail パネルの表示変更

**Before（設定値の羅列）:**
```
Techniques
✓ shadow_retrieval  ✓ clarification  ✗ multi_query ...
```

**After（動作結果）:**
```
動作結果
検索エンジン: Vertex AI Search
検索ヒット: 4件 → リランク後: 4件
聞き返し: なし
権限拒否: なし
```
