## 目的

プロジェクトの設計情報を体系的にドキュメント化し、人間にも Claude にも伝わる形で整備する。
ドキュメントは一度作って終わりではなく、コード変更時に同期メンテされる仕組みを作る。

## 背景・課題

- DD-009-3 で TypeScript UI + Cloud Functions 構成に移行したが、全体像を把握できるドキュメントがない
- CLAUDE.md がテンプレートのままで、プロジェクト固有の技術スタック・規約が未記入
- 新しい会話で Claude を起動するたびに、コードベースの探索から始まる非効率
- DB定義、画面設計、環境構成が暗黙知のままになっている

## 設計方針

### Claude に伝わるドキュメントとは

1. **CLAUDE.md をハブにする** — プロジェクト概要 + 各ドキュメントへのポインタ。新しい会話の最初に読み込まれる
2. **doc/design/ に詳細を置く** — 必要なときだけ参照する設計ドキュメント群
3. **mermaid で図を書く** — テキストベースなので git diff で変更が追える。Claude も読める
4. **コードと近い場所に置く** — 乖離しにくい

### 継続メンテの仕組み

- DDの「仕様書同期」ステップ（Full フロー Step 8）で、関連ドキュメントの更新を必須にする
- CLAUDE.md に「ドキュメント更新ルール」を明記し、Claude が自発的に更新を提案できるようにする
