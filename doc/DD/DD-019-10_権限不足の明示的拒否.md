# DD-019-10: 権限不足の明示的拒否（security 0%→100%）

| 作成日 | 更新日 | ステータス |
|--------|--------|------------|
| 2026-03-22 | 2026-03-22 | 完了（部分的改善: 0%→20%） |

## 目的

権限フィルタリングで除外された文書が存在する場合に、「権限がありません」と明示的に拒否する。security カテゴリのスコアを 0% → 100% に改善し、Overall +6〜7pt を狙う。

## 背景・課題

DD-019-5 で権限フィルタリング（Pre-filtering）を実装済み。Firestoreの `allowed_groups` による検索前フィルタは正常動作しているが、LLMが「権限不足」と回答しない。

### 根本原因

Pre-filteringは検索前に透過的にフィルタするため、LLMからは「権限で除外された」のと「そもそも文書が存在しない」の区別がつかない。プロンプトに `permission_hint` を入れているが効果不十分。

### 外部LLMの分析結果（Gemini / GPT-4o 一致）

> 「RAGの失敗」ではなく「**状態伝達の失敗**」。
> フィルタなし/あり検索の差分を検出し、LLMを呼ばずアプリ側で即拒否するのが最も堅牢。

詳細: [DD-019/llm_response_summary.md](DD-019/llm_response_summary.md)

## 検討内容

### アプローチ: Shadow Retrieval（フィルタなし/あり検索の差分検出）

```
Query
  ↓
① フィルタなし検索（shadow） → スコアのみ取得、本文はLLMに渡さない
② フィルタあり検索（通常）
  ↓
③ 差分判定
  ├─ shadow有り & 通常なし → FILTERED_BY_PERMISSION → アプリ側で即拒否
  ├─ 両方なし → NO_MATCH → 通常フロー（「情報がありません」）
  └─ 通常有り → OK → 通常フロー
```

### 変更対象

| ファイル | 変更内容 |
|---|---|
| `src/search/flow.py` | shadow検索 → 差分判定 → 即拒否の分岐追加 |
| `src/search/retriever.py` | フィルタなし検索関数を追加（or 既存関数にフィルタ無効オプション） |

### 設計ポイント

1. **LLMを呼ばない**: `FILTERED_BY_PERMISSION` 時はアプリ側で固定メッセージを返す（最も堅牢）
2. **本文を渡さない**: shadow検索はスコア/doc_idのみ保持。権限外文書の本文はLLMに渡さない
3. **permission_filter OFF 時**: shadow検索をスキップ（従来動作）
4. **既存の `permission_hint` プロンプト**: 不要になるため削除
