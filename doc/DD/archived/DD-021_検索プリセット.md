# DD-021: 検索プリセット（技術セット切替 + ログ記録）

| 作成日 | 更新日 | ステータス |
|--------|--------|------------|
| 2026-03-22 | 2026-03-22 | 破棄 |

> **番号変更**: DD-019-14 → DD-021 に移動。スコア改善（DD-019系）ではなくUI/運用設計のため独立DD化。

## 目的

1. 検索技術の個別ON/OFFではなく、**プリセット（技術セット）**を定義し、ユーザーはプリセットを選ぶだけで適切な技術の組み合わせが適用されるようにする
2. クエリログにどのプリセット（技術セット）が使われたかを記録し、回答の再現性を担保する

## 背景・課題

### 現状

チャット画面のサイドバーに検索技術のトグルが並んでいる（`ui/src/App.tsx`）:

| 技術 | configKey | 現在のデフォルト |
|------|-----------|----------------|
| ベクトル検索 | (常時ON) | ON |
| ハイブリッド検索 | `hybrid_search` | ON |
| リランキング | (常時ON) | ON |
| メタデータスコアリング | `metadata_scoring` | ON |
| 曖昧質問の聞き返し | `clarification` | ON |
| 権限フィルタ | `permission_filter` | ON |
| 権限除外検出（Shadow） | `shadow_retrieval` | ON |
| マルチクエリ展開 | `multi_query` | OFF |
| 文脈説明の自動付与 | `contextual_retrieval` | ON（再Ingest要） |

### 問題点

- **ユーザーが何を選べばいいか分からない** — 9個のトグルを自由にON/OFFできても、どの組み合わせが適切か判断できない
- **ログに技術状態が記録されない** — `query_logs` に `query`, `answer`, `model`, `elapsed_ms` しかなく、どの技術がONだったか不明
- **評価と実運用の乖離** — 評価はデフォルト設定で実行するが、ユーザーが別の設定で使うと結果が異なる可能性

### 解決方針: プリセット

個別トグルの代わりに、目的別のプリセットを事前定義する:

| プリセット | 想定用途 | 特徴 |
|-----------|---------|------|
| **標準** | 通常の質問応答 | デフォルト設定。バランス型 |
| **高精度** | 正確な回答が必要な場合 | マルチクエリON、全機能フル稼働 |
| **高速** | 素早く概要を知りたい場合 | リランキング省略、メタデータスコアリングOFF |
| **セキュア** | 機密情報を扱う場合 | 権限フィルタ + Shadow Retrieval 必須 |

※ プリセットの具体的な内容は DD-019-10〜13 のフル評価結果を踏まえて確定する。

## 検討内容

### プリセットの定義場所

`src/config.py` または専用ファイル（`src/presets.py`）に辞書で定義:

```python
PRESETS = {
    "standard": {
        "hybrid_search": True,
        "metadata_scoring": True,
        "clarification": True,
        "permission_filter": True,
        "shadow_retrieval": True,
        "multi_query": False,
    },
    "high_accuracy": {
        "hybrid_search": True,
        "metadata_scoring": True,
        "clarification": True,
        "permission_filter": True,
        "shadow_retrieval": True,
        "multi_query": True,
    },
    ...
}
```

### クエリログの拡張

`main.py` `_save_query_log()` に追加:

```python
{
    "query": query,
    "answer": answer,
    "model": model,
    "collection": config.collection_name,     # DD-019-9-2 で追加
    "preset": preset_name,                     # 本DD で追加
    "techniques": {                            # 本DD で追加
        "hybrid_search": True,
        "metadata_scoring": True,
        ...
    },
    "elapsed_ms": elapsed_ms,
    ...
}
```

### UI変更

チャット画面のサイドバー: 個別トグル群 → プリセット選択ドロップダウン

```
┌─ 検索モード ──────────────────┐
│  [標準 ▼]                     │
│    標準 — バランスの良い検索    │
│    高精度 — マルチクエリで徹底  │
│    高速 — リランキング省略     │
│    セキュア — 権限チェック強化  │
│                               │
│  適用中の技術:                 │
│  ✓ ベクトル検索               │
│  ✓ ハイブリッド検索           │
│  ✓ リランキング               │
│  ✓ メタデータスコアリング     │
│  ✓ 曖昧質問の聞き返し        │
│  ✗ マルチクエリ展開           │
└───────────────────────────────┘
```

プリセットを選ぶと、下に適用される技術一覧が表示される（読み取り専用）。
