## 目的

Tuning画面にパラメータの説明・操作フローのガイドを追加し、History画面に2つの評価結果を並べて比較する機能を追加する。チューニング作業を初見でも迷わず実行できるようにする。

## 背景・課題

- Tuning画面のパラメータ（Chunk Size, Top K 等）が何を意味するか、画面上に説明がない
- 「どの順番で何をすればよいか」の操作フローが不明
- 評価結果がHistoryに蓄積されるが、2つの結果を並べて「何を変えたら何が変わったか」を見る手段がない

## 検討内容

### ガイド追加（Tuning画面）

| パラメータ | 説明 | 調整の方向性 |
|-----------|------|-------------|
| Chunk Size | ドキュメントを何文字ずつに分割するか | 大きくすると文脈が増えるが、検索精度が下がる |
| Chunk Overlap | 分割時の重複文字数 | 大きくすると境界のロストが減るが、チャンク数が増える |
| Top K | ベクトル検索で取得する候補数 | 大きくすると検索漏れが減るが、ノイズも増える |
| Rerank Top N | リランキング後に残す件数 | LLMに渡すソース数を制御 |
| Rerank Threshold | リランキングのスコア閾値 | 高くすると精度重視、低くすると再現率重視 |

操作フロー:
1. パラメータを変更
2. 「Re-tune」ボタン（Save → Ingest → Evaluate を一括実行）
3. History画面で前回との比較

### 評価比較機能（History画面）

- 2つの結果にチェックボックス → 「Compare」ボタン
- 比較表示: パラメータの差分 + 各TypeのスコアΔ（増減）
- 変更なしのパラメータはグレーアウト、変更ありはハイライト

## 決定事項

1. **ガイド表示方式**: ツールチップではなくインライン表示（各パラメータ下に1行の説明文）
2. **操作フロー**: 画面上部に1行のインフォボックスで表示
3. **説明の粒度**: パラメータの意味のみ（調整方向性は画面には含めない）
