# DD-009-4: 管理者向けUI（ドキュメント管理・チューニングダッシュボード）

| 作成日 | 更新日 | ステータス |
|--------|--------|------------|
| 2026-03-20 | 2026-03-20 | 進行中 |

## 目的

文書のアップロード、インジェスト結果の確認、評価の実行・結果表示、パラメータチューニングを行える管理者向けUIを構築する。現在のCLI操作（`scripts/ingest.py`, `scripts/evaluate.py`）を画面化し、非エンジニアの管理者でも運用できるようにする。

## 背景・課題

- 現在の文書取り込みは `uv run python scripts/ingest.py` のCLI実行が必要
- 評価も `uv run python scripts/evaluate.py` で手動実行
- 本番構成で「GCSにファイルを置くだけ」は管理として雑
- 管理者が「文書を投入したら結果を確認し、チューニングし直す」サイクルを回せるUIが必要
- DD-009-3でTypeScript移行を決定しており、同一プロジェクトで管理画面も構築できる

## 検討内容

### 管理画面で提供する機能

| # | 機能 | 現状（CLI） | 管理画面で |
|---|------|-----------|-----------|
| 1 | **文書アップロード** | `test-data/sources/` にファイルを手動配置 | ブラウザからドラッグ&ドロップでアップロード |
| 2 | **インジェスト実行** | `scripts/ingest.py` を手動実行 | ボタンクリックで実行。進捗表示 |
| 3 | **インジェスト結果確認** | CLI出力を目視 | チャンク数、メタデータ一覧、分割プレビュー |
| 4 | **評価実行** | `scripts/evaluate.py` を手動実行 | ボタンクリックで実行 |
| 5 | **評価結果表示** | `results/` のJSONを目視 | タイプ別スコア表、失敗ケース詳細、前回比較グラフ |
| 6 | **パラメータ調整** | `src/config.py` を手動編集 | 画面上でチャンクサイズ・top_k・閾値等を変更 |
| 7 | **再チューニング** | パラメータ変更→ingest→evaluate を手動 | パラメータ変更→ワンクリックで再インジェスト→再評価→スコア比較 |

### 画面構成（案）

```
管理画面
├── ダッシュボード
│   ├── 現在のスコア（全体 + タイプ別）
│   ├── 前回スコアとの差分
│   └── 登録文書数・チャンク数
├── 文書管理
│   ├── 文書一覧（ファイル名、カテゴリ、チャンク数、更新日）
│   ├── 文書アップロード（ドラッグ&ドロップ）
│   └── インジェスト実行・進捗表示
├── 評価
│   ├── 評価実行ボタン
│   ├── タイプ別スコア表
│   ├── 失敗ケース詳細（質問、期待、実際の回答）
│   └── 評価履歴（日時順）
└── チューニング
    ├── パラメータ設定（チャンクサイズ、overlap、top_k、rerank_threshold等）
    ├── 再チューニング実行（ingest→evaluate ワンクリック）
    └── スコア推移グラフ
```

### アーキテクチャ

- DD-009-3のTypeScriptフロントエンドと同一プロジェクト
- 利用者向け（チャット）と管理者向け（ダッシュボード）はルーティングで分離
- バックエンドAPI: DD-009-3で構築するAPI + 管理系エンドポイント追加
  - `POST /api/ingest` — インジェスト実行
  - `POST /api/evaluate` — 評価実行
  - `GET /api/evaluate/results` — 評価結果取得
  - `PUT /api/config` — パラメータ更新
  - `POST /api/documents/upload` — 文書アップロード
  - `GET /api/documents` — 文書一覧
