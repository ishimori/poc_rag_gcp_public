## 検討内容

### S1: Shadow Retrieval の改善

security-002, 003 は権限外文書の内容をそのまま回答している。DD-019-10 で Shadow Retrieval を実装済みだが、効いていない可能性がある。

**仮説**: hybrid_search が結果を返しているため `len(search_results) == 0` の条件に入らない。権限外文書が wikipedia 経由で検索結果に混入している可能性。

**対策**: Shadow Retrieval の条件を見直す or rerank 後に権限チェックを追加。

### S2: キーワード検索の強化

「情報なし」の5件は全て、口語的なクエリ（「ソフトを入れたい」「タクシー代を請求」等）と文書内の正式表現（「ソフトウェア申請」「タクシー利用規定」等）の乖離が原因。DD-019-8 と同系統の検索精度問題。

**対策候補**:
- クエリ拡張（LLMでクエリを言い換え → 複数クエリで検索）
- キーワード辞書の追加（口語→正式表現のマッピング）

### S3: 曖昧判定プロンプト調整

ambiguous-005（「期限はいつまで？」）は DD-019-13 で曖昧判定を強化したが、このケースは漏れている。clarifier プロンプトに「対象が特定できない質問」のパターンを追加。

### S4: wikipedia ノイズ対策

semantic-008 は wikipedia のマルウェア記事から回答。社内FAQに同内容があるはずだが、wikipedia チャンクの方がスコアが高い。

**対策候補**:
- カテゴリフィルタ（wikipedia ソースの優先度を下げる）
- メタデータスコアリングの強化

## 決定事項

- **S0（新規）: Ingest バグ修正が最優先** — `chunker.py` の YAML パーサーがブロック形式リストを解析できず、`meeting_minutes_exec.md` の `allowed_groups` が `["all"]` になっている
- **再 Ingest は chunk_size=1200** で実施（DD-019-9 で最高スコア。config.py のデフォルトは 800 のまま維持、DD-022 でコレクション切替を正式導入予定）
- S1（Shadow Retrieval 改善）は S0 修正後に再評価。バグ修正だけで security が改善する可能性
