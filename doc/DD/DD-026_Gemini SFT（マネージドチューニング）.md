## 目的

Vertex AI のマネージドチューニング（Supervised Fine-Tuning）を使い、RAGの回答品質をチューニングする。Search Tuning（負例10,000件必要、見送り済み）とは異なり、prompt + 理想的な回答のペア約100件で実施可能。

## 背景・課題

- Search Tuning は負例10,000件のハードルが高く見送り（`vertex-ai-search-comparison.md` に記録済み）
- マネージドチューニング（SFT）は**最小100件**のトレーニングデータで実施可能
- 本PoCの `eval_dataset.jsonl` に74件の query + expected_answer がある → 30件追加で100件到達
- 改善対象は「検索ランキング」ではなく「LLMの回答品質」（形式統一、拒否判断、権限応答等）

## 検討内容

### Search Tuning vs SFT

| 観点 | Search Tuning | SFT（本DD） |
|------|:---:|:---:|
| 何をチューニング？ | 検索ランキング | **LLMの回答品質** |
| データ形式 | query + document + label | **prompt + 理想的な回答** |
| 最小データ量 | 10,000件以上 | **約100件** |
| 改善対象 | 検索結果の関連性 | 回答の形式・正確性・拒否判断 |

### トレーニングデータの形式

```json
{"messages": [
  {"role": "user", "content": "ネジ999999の材質は？"},
  {"role": "model", "content": "SUS304（オーステナイト系ステンレス鋼）です。"}
]}
```

### 期待される改善領域

| 領域 | 現状の問題 | SFTで改善可能か |
|------|----------|:-:|
| 回答形式の統一 | 回答の長さ・トーンがバラバラ | ✅ |
| 「記載なし」の判断 | 無関係な情報から無理に回答するケースあり | ✅ |
| 権限不足時の応答 | 「記載がありません」と「権限がありません」の使い分け | ✅ |
| 表データの読み取り | 行列交差の抽出ミス | △（データ依存） |
