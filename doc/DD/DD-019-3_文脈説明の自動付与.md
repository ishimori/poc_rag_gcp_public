# DD-019-3: 文脈説明の自動付与（#10 Contextual Retrieval）

| 作成日 | 更新日 | ステータス |
|--------|--------|------------|
| 2026-03-21 | 2026-03-21 | 進行中 |

## 目的

Ingest時にLLMで各チャンクに固有の文脈説明を自動生成し、検索精度を底上げする。ヘッダーインジェクション（#02）の上位互換。

## 背景・課題

現在のヘッダーインジェクション（#02）は全チャンクに同一の文書タイトルを付与するだけ:

```
[VPN接続マニュアル]
同社の接続成功率は前四半期比3%増加しました。
```

この固定ラベルでは、チャンクが文書のどの部分にあるか、何について述べているかの情報が欠落している。

Contextual Retrievalでは、LLMが文書全体を読んでチャンク固有の説明を生成:

```
[VPN接続マニュアル > 第5章 運用実績 > 2024年度の接続成功率推移データ]
同社の接続成功率は前四半期比3%増加しました。
```

### 期待される効果

| 組み合わせ | 検索失敗率の削減 |
|-----------|----------------|
| Contextual Retrieval 単体 | 35% |
| Contextual Retrieval + リランキング（実装済み） | **67%** |

（出典: Anthropic ベンチマーク）

### 制約

- **再Ingestが必要**: 全チャンクのEmbeddingを再生成する
- **Ingest時間が増加**: 各チャンクごとにLLM呼び出しが発生
- **コスト**: Gemini 2.5 Flash + Context Caching で数百円/1,000チャンク

## 検討内容

### ヘッダーインジェクションとの関係

研究資料で「上位互換」と位置づけられている。実装方針:

| 案 | 内容 | メリット | デメリット |
|----|------|---------|-----------|
| A: 置換 | contextual_retrieval ON → header_injection OFF | シンプル | 切替時にA/Bテストしにくい |
| **B: 独立制御** | 両方を独立にON/OFF可能 | アブレーション可能 | 両方ONの場合の挙動定義が必要 |

**→ 案B採用**: `config.contextual_retrieval`を追加。ONの場合はLLM生成の文脈説明をプレフィックスに使用し、header_injectionのプレフィックスは付けない（contextual_retrievalが優先）。

### Ingestパイプラインへの統合

現在:
```
chunk_document() → embed_texts() → store_chunks()
```

変更後:
```
chunk_document() → contextualize_chunks() → embed_texts() → store_chunks()
```

`contextualize_chunks()`は新規モジュール`src/ingest/contextualizer.py`に実装。

### LLMプロンプト設計

Anthropicの推奨プロンプトを参考に、日本語で設計:

```
以下の文書の一部（チャンク）に、簡潔な文脈説明を付けてください。
文脈説明は、このチャンクが文書のどの部分にあり、何について述べているかを1〜2文で説明するものです。

## 文書全体
{document_text}

## 対象チャンク
{chunk_text}

## 出力形式
文脈説明のみを出力してください。他のテキストは不要です。
```

### Context Caching

同一文書の複数チャンクで文書全体テキストを繰り返し送信するため、Gemini Context Cachingを使うとコスト90%削減が見込める。ただしPoCでは文書数が少ないため、まずはCachingなしで実装し、効果を確認する。

## 決定事項

- **独立制御**: `config.contextual_retrieval`で切替。ONの場合header_injectionより優先
- **新規モジュール**: `src/ingest/contextualizer.py`
- **LLMモデル**: Gemini 2.5 Flash（config.llm_modelを流用）
- **Context Caching**: PoCでは不使用。本番化時に導入検討
- **再Ingest**: 実装後に`python scripts/ingest.py --clear`で全データ再投入
