## 目的

Firestoreに格納されたチャンクデータを可視化するブラウザページを追加する。RAGの入力データがどういう状態かを人間が確認できるようにする。

## 背景・課題

- インジェスト済みのデータが30チャンクあるが、中身を確認するにはFirestoreコンソールに行くしかない
- チャンク分割の結果やメタデータの付与状況を一覧で確認したい
- ただしFirestoreは1読み込みごとに課金されるため、毎回リアルタイム読み込みは避けたい

## 検討内容

### データ取得フロー

```
[Firestoreデータブラウザ画面]

初回 or データ更新時:
  「データをダウンロード」ボタン → Firestoreから全件取得 → ローカルCSV保存

通常閲覧時:
  ローカルCSVから読み込み → テーブル表示
```

### 画面構成

```
+------------------------------------------+
| 📊 データブラウザ                          |
+------------------------------------------+
| [データをダウンロード] [最終取得: 03/20 15:30] |
|                                          |
| コレクション: chunks (30件)                |
|                                          |
| | source_file | chunk_index | category | security_level | content (先頭50文字) |
| |-------------|-------------|----------|----------------|---------------------|
| | parts_catalog.md | 0 | parts | public | [部品カタログ]... |
| | parts_catalog.md | 1 | parts | public | [部品カタログ]... |
| | ...         |             |          |                |                     |
|                                          |
| ▼ チャンク詳細（行クリックで展開）          |
| content: [部品カタログ]\n部品番号: 999999... |
| embedding: [0.012, -0.034, ...] (768次元)  |
+------------------------------------------+
```

### ページ構成（st.navigation）

| ページ | パス | 内容 |
|--------|------|------|
| チャット | `features/chat/index.py` | 既存のRAGチャットUI |
| データブラウザ | `features/data_browser/index.py` | Firestoreデータの閲覧 |

※ `pages/` はStreamlitの自動マルチページ検出と競合するため使わない

## 決定事項

- `st.navigation` で複数ページ構成にする
- FirestoreデータはローカルCSV経由で表示（コスト削減）
- CSVは `data_cache/` に保存（gitignore対象）
- embeddingカラムはCSVに含めない（サイズが大きすぎる）
