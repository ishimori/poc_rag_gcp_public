## プロンプト

### プロジェクト概要

- **PoCフェーズ**: ローカル開発環境で検証中

### 残っている課題（スコアが低いカテゴリ）

#### 1. security 0% (0/5) — 最大の課題

**症状**: Firestoreの Pre-filtering（`allowed_groups` による検索前フィルタ）は正常動作している。権限外の文書はそもそも検索結果に含まれない。しかし、LLMが「権限不足のため回答できません」と明示的に回答せず、「該当する情報が見つかりませんでした」と回答する。テストケースの期待回答は「権限がありません」系の拒否文言。

**根本原因**: LLMは「検索結果にない」のと「権限で除外された」の区別がつかない。Pre-filteringは検索前に透過的にフィルタするため、LLMにはフィルタされたことが見えない。

#### 2. semantic 67% (8/12) — 改善余地あり

**失敗パターン**: 意味的に関連する文書を引けない、または引けても正確に回答できないケースが4件残存。Embedding task_type指定とキーワード検索追加で17%→67%まで改善したが、まだ4件失敗。

#### 3. ambiguous 60% (3/5) — LLM判定のぶれ

**症状**: 聞き返し機能自体は動作するが、LLM-as-Judgeの判定にぶれがある。同じ質問を複数回評価すると結果が変わることがある。テストケース設計の問題も含む可能性。

#### 4. unanswerable 80% (4/5) — 過剰回答

**症状**: 文書に答えがない質問に対して、1件だけLLMが無理に回答を生成してしまう。

### アーキテクチャ概要

```
ユーザークエリ
  ↓
[Pre-RAG] 曖昧判定（Clarifier） → 曖昧なら聞き返し
  ↓
[Pre-RAG] AIフィルタ自動生成 → クエリからフィルタ条件を抽出
  ↓
[検索] ハイブリッド検索
  ├─ ベクトル検索（Firestore Vector Search, Pre-filtering付き）
  └─ キーワード検索（型番正規表現 + 一般語テキストマッチ）
  ↓
[後処理] RRF統合 → リランキング（Vertex AI Ranker） → メタデータスコアリング
  ↓
[生成] Gemini 2.5 Flash で回答生成
```
