## 目的

GCP環境を構築し、「コードを変えたら自動でスコアが出る」開発サイクルを実現する。Claude Codeが実装→テスト→評価→改善を効率的に回せる仕組みを作る。

## 背景・課題

- テストデータ（DD-007）は揃ったが、それを流すパイプラインがない
- GCP環境が未構築のため、Firestore / Embedding API / Ranking API を試せない
- チューニングの効果を「数値で」確認できる仕組みがないと、改善が感覚頼みになる

### 自動化パイプライン

3つのスクリプトで構成する。

#### 1. `scripts/ingest.ts` — インジェスト

```
test-data/sources/*.md → チャンク分割 → Embedding → Firestoreに格納
```

- 入力: `test-data/sources/` のMarkdownファイル
- 処理: チャンク分割（Recursive, サイズ/オーバーラップはパラメータ化）→ ヘッダーインジェクション → gemini-embedding-001でベクトル化 → Firestoreに書き込み
- メタデータ: `category`, `security_level`, `allowed_groups`, `source_file`, `chunk_index` を自動付与
- 冪等性: `content_hash` による重複防止

#### 2. `scripts/search.ts` — 検索（RAG Flow）

```
質問テキスト → クエリ拡張 → Firestore検索 → リランキング → 回答生成
```

- Genkit Flowとして実装
- パラメータ: チャンクサイズ、検索件数(top-k)、リランキング閾値、LLMモデル
- 各ステップのログ（検索結果、スコア、最終回答）をJSON出力

#### 3. `scripts/evaluate.ts` — 評価

```
eval_dataset.jsonl → 各質問をRAG Flowに投げる → 回答を自動採点 → スコアレポート出力
```

- 判定ロジック:
  - `expected_keywords` の含有チェック（キーワードベース）
  - Genkit Evaluator の Faithfulness / Answer Relevancy（LLM判定）
- 出力: タイプ別スコア + 全体スコア + 失敗ケースの詳細

### 開発サイクル

```
パラメータ変更（チャンクサイズ等）
    ↓
$ npm run ingest     # ソースドキュメントを再インジェスト
    ↓
$ npm run evaluate   # 45件のQ&Aで自動評価
    ↓
スコアレポート出力（前回比較）
    ↓
スコアが下がったケースを分析 → 次の改善
```

### プロジェクト構成

```
poc_rag_gcp_public/
├── src/
│   ├── config.ts          # パラメータ設定（チャンクサイズ等）
│   ├── ingest/
│   │   ├── chunker.ts     # チャンク分割ロジック
│   │   ├── embedder.ts    # Embedding API呼び出し
│   │   └── store.ts       # Firestore書き込み
│   ├── search/
│   │   ├── flow.ts        # Genkit RAG Flow
│   │   ├── retriever.ts   # Firestore検索
│   │   └── reranker.ts    # Ranking API呼び出し
│   └── evaluate/
│       ├── runner.ts       # 評価実行
│       ├── scorer.ts       # スコアリングロジック
│       └── reporter.ts     # レポート出力
├── scripts/
│   ├── ingest.ts           # npm run ingest のエントリポイント
│   ├── search.ts           # npm run search のエントリポイント
│   └── evaluate.ts         # npm run evaluate のエントリポイント
├── test-data/              # DD-007で作成済み
├── results/                # 評価結果の保存先（gitignore）
└── package.json
```

### やらないこと

- フロントエンド（UIは後で。まずはCLIで完結させる）
- CI/CD（ローカルで手動実行。安定したら自動化を検討）
- 本番デプロイ（ローカル開発環境のみ）

## 決定事項

- GCPプロジェクトを新規作成し、Firestore / Vertex AI / Discovery Engine を有効化
- Node.js + Genkit + TypeScript で実装
- `npm run ingest` → `npm run evaluate` の2コマンドで評価サイクルが回る設計
- 評価結果は `results/` に日時付きJSONで保存
