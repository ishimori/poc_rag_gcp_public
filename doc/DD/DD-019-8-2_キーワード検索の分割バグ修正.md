# DD-019-8-2: キーワード検索の分割バグ修正

| 作成日 | 更新日 | ステータス |

## 目的

`keyword_searcher.py` の `_extract_keywords()` が漢語複合語を分割できないバグを修正し、temporal テストケース等の検索精度を改善する。

## 背景・課題

DD-019-8 で追加した一般語キーワード検索（対策2）に設計バグが残っている。

### バグの内容

`_extract_keywords()` は正規表現 `[\u4e00-\u9fff\u30a0-\u30ffA-Za-z]+` で漢字連続を1トークンとして抽出する。日本語の形態素解析を行っていないため、助詞を挟む表現とマッチしない。

```
クエリ: 「有給の繰越上限は何日？」
抽出:   ['有給', '繰越上限', '何日']

文書内: 「繰越の上限は20日とする」
→ 「繰越上限」≠「繰越の上限」 → マッチしない
```

### 影響範囲

DD-019-7 の Phase 1 診断で発覚:
- **temporal-001**（繰越上限は何日？）: 「繰越上限」が「繰越の上限」にマッチせず → leave_policy チャンクが検索結果に含まれない → FAIL
- **temporal-003**（最新の休暇規程はいつ改定された？）: 「休暇規程」は文書内「有給休暇規程」に部分一致するが、汎用キーワードのため wikipedia に埋もれる → FAIL
- temporal 以外にも、助詞挟みの表現で同様の漏れが発生している可能性

### temporal-002 が PASS する理由

「計画年休」は文書内にそのまま存在（「年間5日を**計画年休**として」）→ 分割不要でマッチする。

## 検討内容

### 対策案

| 案 | 概要 | メリット | デメリット |
|---|---|---|---|
| A. 漢語を2文字ずつに分割 | 4文字以上の漢語 → 2文字ずつ分割 | 簡単、即効 | 過剰分割のリスク（「情報セキュリティ」→ 「情報」+「セキ」） |
| B. N-gram 方式 | 2文字 bi-gram を生成 | 網羅的 | マッチ数が爆発、ノイズ増 |
| C. 形態素解析導入 | MeCab/Sudachi 等を使用 | 正確 | 依存関係追加、デプロイ複雑化 |
| D. 漢語のみ2文字分割 + 元トークン保持 | 4文字以上の漢字列を2文字ずつに分割し、元トークンも残す | 部分マッチと完全マッチの両方をカバー | 若干のスコア膨張 |

**案D を採用**: 元トークンを残すことで完全一致の高スコアを維持しつつ、2文字分割で助詞挟みにも対応する。形態素解析の導入は PoC の範囲を超える。

### 具体例

```
入力: 「有給の繰越上限は何日？」
現在: ['有給', '繰越上限', '何日']
修正後: ['有給', '繰越上限', '繰越', '上限', '何日']

→ 文書内「繰越の上限」に「繰越」「上限」が個別にマッチ → スコア加算 → RRF で浮上
```

## 決定事項

- 案D（漢語のみ2文字分割 + 元トークン保持）を採用
- カタカナ語は分割しない（「セキュリティ」等は1語として扱う）
