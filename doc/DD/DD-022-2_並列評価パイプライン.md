## 目的

Evaluate（精度評価）のテストケースを並列実行し、評価時間を短縮する。

## 背景・課題

- 現在の `evaluate.py` は 74 件のテストケースを1件ずつ順次実行（約10分）
- 各テストケースは独立（RAG検索 → リランキング → LLM回答 → 判定）で、依存関係がない
- チャンクサイズ実験（DD-022）で複数コレクションの評価を繰り返すため、高速化の価値が大きい

## 検討内容

### 並列化方式

`concurrent.futures.ThreadPoolExecutor` を採用:
- `run_case()` がスレッドセーフ（各ケースが独立した API 呼び出し）
- GIL の影響は I/O バウンド（API 呼び出し待ち）なので問題なし
- `ProcessPoolExecutor` は `config` のグローバル状態共有が面倒

### 並列数

デフォルト 5 並列。Gemini API のレートリミット次第で調整可能（`--workers` 引数）。

### 結果の順序

`as_completed` で完了順に処理するが、最終レポートは元のケース順に並べ替える。

## 決定事項

- `scripts/evaluate_parallel.py` として新規作成（既存の `evaluate.py` は変更しない）
- `--workers` で並列数を指定可能（デフォルト 5）
- `--collection`, `--limit` は既存 `evaluate.py` と同じインタフェース
- task_status 対応（Operations Monitor で進捗表示可能）
